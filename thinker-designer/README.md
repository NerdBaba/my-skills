# Thinker-Designer Skill

A design philosophy and workflow that transforms an AI from a "Graphic Designer" (making things look nice) into a "Design Thinker" (solving problems through intentionality).

## 🚀 How to Use

Trigger this skill whenever you need to:
- Redesign a website, flyer, or logo.
- Create new visual assets (UI/UX, social media graphics, etc.).
- Get creative direction or visual improvements.

### 🧩 Triggering the Skill
Simply ask for a design-related task:
- *"Can you redesign my website?"*
- *"Help me create a flyer for a local food drive."*
- *"I need a professional and modern look for my AI app."*

The skill will automatically activate and pause to ask critical questions before proposing any visual elements.

## 🛠 Installation & Portability

To use this skill in other projects or on different PCs:

1. **Directory Structure:**
   Store your custom skills in a dedicated folder. A recommended structure is:
   ```text
   your-project/
   ├── .junie/
   │   └── skills/
   │       └── thinker-designer/
   │           ├── SKILL.md       <-- Main logic
   │           ├── README.md      <-- This file
   │           ├── scripts/       <-- Optional (automation scripts)
   │           ├── references/    <-- Optional (case studies, docs)
   │           └── assets/        <-- Optional (templates, icons)
   ```

2. **Moving the Skill:**
   - Copy the `thinker-designer` folder to your new PC or project.
   - Place it in the `.junie/skills/` directory (or any directory where your AI agent is configured to look for skills).

3. **Activation:**
   - Ensure your AI agent (like Junie or Claude with MCP) is aware of this directory.
   - In most Junie-compatible environments, the agent will automatically detect and read the `SKILL.md` file within your project's `skills` folder.

## 🧠 The Thinker's Protocol

1. **Stop and Ask "Why?":** Identify the audience, goal, core problem, and story.
2. **Problem Statement:** Reframe the design task as a problem to be solved.
3. **Content First:** Draft the core message before the visuals.
4. **Intentional Logic:** Every design choice (color, font, layout) must have a strategic reason.
